# فورک در گیتهاب
## یعنی ساختن یک نسخه ی شخصی از repository  دیگران در حساب گیتهاب فرد و تغییرات بدون تاثیر روی پروژه ی اصلی اعمال میشود
## مزیت:
1.اگر فرد میخواهد تغییراتی روی پروژه اصلی اعمال کند اول ان را فورک میکند
2.بدون خراب شدن پروژه ی اصلی فرد میتواند ویژگی های جدیدی را بررسی کند و با pull request ان را به پروژه اصلی اضافه کند
